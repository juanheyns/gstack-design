UI mockup CLI

Installation

Homebrew (recommended)

brew tap juanheyns/gstack
brew install design
design setup   # configure your OpenAI API key

Manual

# Download a release binary from GitHub and put it in your PATH
curl -L https://github.com/juanheyns/gstack-design/releases/latest/download/design-darwin-arm64.tar.gz | tar xz
mv design /usr/local/bin/
design setup

Build from source

git clone https://github.com/juanheyns/gstack-design.git
cd gstack-design
./setup              # requires bun >= 1.0.0
dist/design --help

Claude Code skill (skills.sh)

npx skills add juanheyns/gstack-design --skill gstack-design
# Installs the design skill into your AI agent for use in any project.

Quick start

Generate a mockup

design generate --brief "Dashboard with metrics cards and a sidebar"
# Output: .design/mockup.png

design generate --brief "Landing page for a dev tool" --check
# Generate + auto-QA: verifies the mockup matches the brief

Explore variants

design variants --brief "Settings page with toggles" --count 3
# Output: .design/variants/variant-A.png, variant-B.png, variant-C.png

design compare --images ".design/variants/*.png" --serve
# Opens an interactive comparison board in your browser

Iterate on a design

design iterate --session .design/variants/session.json \
  --feedback "Make the header smaller, add more whitespace"
# Multi-turn refinement with conversation threading

Evolve a live site

# Screenshot your current site (via browse or any tool)
browse goto https://example.com
browse screenshot /tmp/current.png

# Generate an evolved version
design evolve --screenshot /tmp/current.png \
  --brief "simplify the header, add more whitespace"

# Diff to see what changed
design diff --before /tmp/current.png --after .design/evolved.png

Design-to-code handoff

design design-to-code --image .design/mockup.png
# Returns: { implementationPrompt, colors, typography, layout, components }

Command reference

Generation

design generate --brief "..."Generate a mockup from a text brief
design generate --brief-file brief.jsonGenerate from a structured JSON brief
design generate --brief "..." --checkGenerate + auto-QA the result
design variants --brief "..." --count 3Generate N variant explorations
design iterate --session ... --feedback "..."Refine a mockup with feedback

Quality & comparison

design check --image ... --brief "..."Vision QA — does mockup match brief?
design compare --images "*.png"HTML comparison board
design compare --images "*.png" --serveComparison board + live feedback server
design diff --before a.png --after b.pngVisual diff between two mockups
design verify --mockup ... --screenshot ...Verify implementation against mockup

Design evolution

design evolve --screenshot ... --brief "..."Evolve a live screenshot into a new design
design design-to-code --image ...Extract implementation prompt from mockup
design extract --image ...Extract design language into DESIGN.md

Gallery & server

design galleryHTML timeline of all design explorations
design serve --html board.htmlServe comparison board with feedback loop
design setupConfigure OpenAI API key
design --versionPrint version

Agent workflow

Every command outputs structured JSON to stdout and logs to stderr, making design ideal for AI agent automation.

generate → check → iterate → compare → design-to-code → implement

Project directory

All output defaults to .design/ in the current git project root. Auto-created on first use, auto-added to .gitignore.

.design/
  mockup.png — latest generate output
  variants/
    variant-A.png
    variant-B.png
    variant-C.png
  board.html — comparison board
  iterate.png — latest iteration
  evolved.png — latest evolve output
  gallery.html — design history timeline

Structured brief format

// brief.json — for --brief-file
{
  "goal": "Dashboard for coding assessment tool",
  "audience": "Technical users, hiring managers",
  "style": "Dark theme, cream accents, minimal",
  "elements": ["score badge", "narrative letter", "metric cards"],
  "screenType": "desktop-dashboard"
}

design setup                    # interactive API key setup
export OPENAI_API_KEY=sk-...    # or set via environment variable
# Config stored at ~/.config/design/config.json
# Also reads ~/.gstack/openai.json as fallback for gstack users
